1. Config file - how to configure your reverse proxy


->status
    format - status=0/1

    0 refers to off and 1 refers to on. In off mode, every request will
    be rejected, regardless of the status of the main/target server.


->port
    format - port=1234   (number of the port)

    It is the local port number on which the proxy will operate


->target
    format - target=1234

    It is the local port number on which the main/target server runs on


->cipherkey
    format - cipherkey=0123456789abcdefghijklmnopqrstuv    (string of 32 characters)

    It is the cipher key used for AES-256-GCM encryption of cookies. It
    uses a 32 bytes buffer, which is derived from this cipherkey string.
    Make sure you use a strong encryption key with use of mix of alphabets,
    special characters and numbers.
    This cipher key should not be leaked or shared, or else security can be breached


->cookieEncryptOption
    format - cookieEncryptOption=0/1

    0 refers to no and 1 refers to yes. Enabling this will make the value of the
    cookies coming from the target server encrypted and stay in the encrypted format
    in the user's browser. However, the target server will receive cookie values in plain
    text format in forwarded requests.


->maxRequestRateLimit
    format - maxRequestRateLimit=1000

    This reverse proxy uses a ratelimiter which limits the rate of incoming
    requests on 2 basis: requests per second and max memory allowed to be used. This value
    is the max requests per second. If you dont know what to type in here, refer to
    the list below.

        10 users - 200
        50 users - 1000
        100 users - 2000
        200 users - 4000
        500 users - 10000
        (20 per user, for an average website, can vary depending on your target server)


->inspectOriginMode
    format - inspectOriginMode=blacklist/whitelist

    NetBreaker offers 2 forms of traffic blocking, on the basis of IP addresses.
    Whitelisting will only allow incoming traffic from IP addresses explicitly
    mentioned in ipWhiteList.txt & Blacklisting allows all incoming traffic
    except IP addresses explicitly mentioned in ipBlackList.txt

    By default NetBreaker uses IP whitelisting. If you want to use IP blacklisting,
    type "blacklist" as the value. Any other value beside it will not be interpreted,
    and NetBreaker will use the default setting (whitelist).
